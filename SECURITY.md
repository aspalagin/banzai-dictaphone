# Безопасность

## Поддерживаемая версия

Исправления безопасности вносятся в ветку `main`.

## Сообщить об уязвимости

Не публикуйте токены, адреса серверов, записи или расшифровки в issue.
Создайте приватный security advisory через вкладку Security репозитория либо
свяжитесь с владельцем репозитория через GitHub. Приложите минимальные шаги
воспроизведения без реальных персональных данных.

## Фактическая модель защиты

HTTP- и WebSocket-эндпоинты требуют `DICTAPHONE_TOKEN`. Токен принимается в
`X-Dictaphone-Token` или как `Authorization: Bearer <token>` и сравнивается
на стороне сервера. `DICTAPHONE_ALLOW_INSECURE=1` отключает эту проверку и
предназначен только для локальной разработки.

`aiohttp`-сервер не завершает TLS: по умолчанию он слушает `127.0.0.1:8097`.
Для доступа по сети завершайте TLS в reverse proxy или туннеле, ограничивайте
доступ к серверу и не передавайте токен по незашифрованному публичному HTTP.

Ключи API и Telegram хранятся в переменных окружения или файлах, на которые
ссылаются переменные `*_FILE`. Не коммитьте их. Файл `.env.example` содержит
только пустые или демонстрационные значения.

Записи, события и транскрипты остаются в каталоге сессий на диске; в проекте
нет автоматической очистки локальных сессий. Оператор отвечает за права на
каталог, резервные копии, срок хранения и удаление. Batch STT создаёт OGG и
загружает его в Yandex Object Storage; объект удаляется после обработки, если
`DICTAPHONE_YANDEX_STT_DELETE_OBJECT` не выключен. При Telegram-доставке
транскрипт и аудио передаются в настроенный Telegram-чат.
